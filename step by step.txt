# **3Ô∏è‚É£ Data Cleaning**

Meliputi:

### ‚úî Handling Missing Values

### ‚úî Outlier Detection

Gunakan:

* IQR method

---

# **4Ô∏è‚É£ Feature Engineering (Tambahan Penting)**

Tahapan ini opsional tapi sangat meningkatkan akurasi model:

### ‚úî Membuat fitur baru seperti:

* Retransmission Rate ‚Üí dari packet loss & throughput
* Delay Variation (jitter) ‚Üí Latency fluctuation
* Throughput Ratio ‚Üí actual / expected

### ‚úî Mengubah Delay-Client & Delay-Server

Jika Delay-Client + Delay-Server > E2E Delay (tidak wajar) ‚Üí tandai error data.

Fitur engineering ini memperkuat model *tree-based, SVM, dan fuzzy logic*.

---

# **5Ô∏è‚É£ Rule-Based Labeling (Penting untuk penelitianmu)**

Labeling dilakukan menggunakan **aturan otomatis** berdasarkan pola berikut:

### Contoh:

```
IF Energy < 20% AND Throughput ‚Üì THEN label = Energi Error
IF Delay ‚Üë AND PacketLoss ‚Üë AND Throughput ‚Üì THEN label = Komunikasi Error
IF E2E Delay tinggi AND PacketLoss rendah THEN label = Routing Error
IF PacketLoss tinggi AND Energy cepat habis THEN label = Loop Error
```

Aturan dibuat dari **tabel indikator** yang sudah kamu susun.

üëâ Ini membuat dataset **konsisten** dan model dapat belajar pola dengan benar.

---

# **6Ô∏è‚É£ Data Balancing (Tambahan yang kamu butuhkan)**

Masalah umum: dataset tidak seimbang (class imbalance).

Solusi:

* **Oversampling** (SMOTE, ADASYN)
* **Undersampling** (random undersampling)
* atau **penyesuaian proporsi di dataset sintetis**

Balancing penting agar **kelas minoritas seperti Loop Error** tidak diabaikan model.

---

# **7Ô∏è‚É£ Normalisasi / Standardisasi**

### ‚úî Normalisasi dipakai untuk:

* SVM

### ‚úî Tidak wajib untuk:

* MinMaxScaler

Catatan: Normalisasi dilakukan **SETELAH** train-test split agar tidak terjadi data leakage.
